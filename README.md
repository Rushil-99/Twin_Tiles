# Twin_Tiles

1. Dynamic Programming (DP)Dynamic Programming is utilized to track and evaluate "Synergy Chains" (consecutive connected tiles with the same pip value) efficiently.Memoization Table: The game uses a 2D array, dpChainTable, to store the longest matching chain ending at any given cell (r, c).State Transition: When a tile is placed, the DP value is calculated by checking its four orthogonal neighbors: calculateDPChain(cell, value) = 1 + max(dpChainTable of matching occupied neighbors).Time Complexity: Because it relies on previously computed overlapping subproblems (the neighbors' DP values), evaluating a new placement takes $O(1)$ time.Rebuilding State: If a move is undone, the DP table is completely rebuilt by replaying the sequence of placed dominoes to ensure accurate state reconstruction.

2. Greedy ApproachThe GreedyStrategy class represents a CPU opponent that makes locally optimal choices to maximize immediate point gains without looking ahead to future turns.Immediate Maximization: The algorithm generates all currently valid moves on the board and scores them individually.Heuristic Evaluation: It selects the move with the highest immediate score by evaluating it through the ScoringHelper.Scoring Priorities: The greedy evaluation is weighted by the base pip sum, the DP chain multiplier, constraint scores (how tight the space is), rarity scores, and isolation penalties (avoiding leaving unplayable gaps).

3. Divide and ConquerThe DivideAndConquerStrategy class demonstrates this paradigm by breaking the board down into smaller sub-problems to find the best possible move.Divide: The algorithm separates the board into four distinct quadrants or zones (Vertical, Even, Odd, Horizontal) and extracts the valid moves available in each.Conquer: It processes the moves from each zone individually, using the classic Merge Sort algorithm to order the moves from lowest to highest score.Combine: The sorted lists from the four zones are merged back together sequentially (mergeZoneLists) to produce a single, fully sorted list of all valid moves, allowing the CPU to instantly pick the highest-scoring option.

4. Minimax with Alpha-Beta PruningThe MinimaxStrategy class implements a game theory search algorithm to create a "Grandmaster CPU" that anticipates future moves.Adversarial Search: The algorithm simulates the game tree up to a specific depth (MAX_DEPTH = 3), alternating between maximizing the CPU's advantage and minimizing the human player's advantage.Alpha-Beta Pruning: It tracks the best guaranteed scores for both players (alpha and beta). If it discovers a branch that leads to a worse outcome than a previously checked branch, it prunes (skips) that subtree entirely, saving computation time.Move Ordering: To make the Alpha-Beta pruning as efficient as possible, moves at each node are pre-sorted using the greedy ScoringHelper heuristic. Checking the most promising moves first leads to earlier and more frequent branch pruning.

5. Graph Theory and RepresentationThe board's structural logic relies heavily on graph data structures to manage spatial relationships and calculate bonuses.Adjacency List: The BoardGraph class models the board as a graph where each cell is a vertex and placed dominoes establish edges between them.Vertex Degree: The graph tracks the connections and can quickly calculate how many occupied neighbors a specific cell has by looking up its edges in the adjacencyList.Game Mechanics: This graph representation directly powers the "Crowd Bonus" scoring feature and helps determine if a proposed move will mathematically isolate other empty cells.
